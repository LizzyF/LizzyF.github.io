Pair Programming Overview

By: Trevor Nelson & Elizabeth Fu

What is pair programming?

Pair programming is a programming technique where two programmers work on the same piece of code together. Generally, this is done by one programmer, the navigator, leading the other programmer, the driver, while the driver enters in the code.

Why is pair programming important?

  • Pair programming is used often in professional programming projects.
  • More accurate code due to error checking.
  • Collaboration produces more efficient code and functionality.
  • Pair programming is often used in the job interviewing process.

Compare Driver and Navigator

The driver is the person typing the code, while the navigator provides guidance on the direction of the code. The driver should only be focused on code that the navigator instructs them to work on.
