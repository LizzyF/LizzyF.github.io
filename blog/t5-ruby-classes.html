Ruby Classes

12/28/14

Classes define the attributes and behaviors of objects. To define a class, one uses the class keyword. The class name should be capitalized. A class is like an outline of what an object's characteristics are, and what it can do. These two things are the class's state and behavior. When an object is created, it is instantiated, and is an instance of the class. So if there is an ice cream class, the flavor chocolate would be an instance of that class.

An instance variable is represented by the @ symbol before the name, and keeps track of the object's state. The instance variable exists as long as the object it is tied to exists. An instance method only applies to one particular instance of a class.

Class variables apply to an entire class, unlike instance variables, and use the double @@ sign. Class methods are exactly that, methods that apply to an entire class and not just one instance, or object, of that class. Constants are variables that never change, and the first letter of the variable name is capitalized, though capitalizing the entire name is the convention.
